¿Qué es ECMAScript?

ECMAScript es un conjunto de reglas que dice cómo debe funcionar un lenguaje de programación. No es un lenguaje en sí, sino un estándar que lenguajes como JavaScript siguen para asegurarse de que todos hagan las mismas cosas de la misma manera. Por ejemplo, si alguien crea un lenguaje basado en ECMAScript, como JavaScript, debe seguir esas reglas para que sea "compatible" y entienda cómo hacer las cosas. El grupo encargado de mantener estas reglas se llama Ecma TC39, y cada cierto tiempo actualizan el estándar para mejorar cosas, agregar nuevas funciones o arreglar problemas. El más famoso es JavaScript, pero también se utiliza en otros lenguajes como JScript y ActionScript. Entonces, cuando usas JavaScript, en realidad estás usando un lenguaje que sigue el estándar ECMAScript. ¡Es como un manual para hacer las cosas de forma ordenada!

Versiones de ECMAScript

Desde su creación, ECMAScript ha evolucionado a través de diferentes versiones. Aquí te presentamos un resumen de las versiones más importantes:

Diferencias Notorias Entre Versiones

A continuación se presentan algunas de las diferencias más destacadas entre las versiones de ECMAScript...

  • ES3 (1999): Fue una actualización importante en su época. Estableció muchas de las bases que usamos en JavaScript hoy en día, como la manipulación de cadenas de texto, manejo de errores, y las funciones básicas.
  • ES5 (2009): Introdujo muchas características importantes que hicieron JavaScript más robusto y fácil de usar:

    • Modo estricto ("use strict") para evitar errores comunes.
    • Métodos nuevos para trabajar con arrays, como .forEach() y .map().
    • Introducción de getter y setter (funciones para acceder y modificar propiedades de objetos).
  • ES6 (2015) ¡La GRAN actualización!: Fue una de las actualizaciones más grandes de JavaScript. Introdujo muchas características nuevas, que hicieron el lenguaje más moderno y fácil de escribir.
    Algunas de las nuevas características incluyen:
    • Let y const (para declarar variables).
    • Funciones flecha (=>).
    • Clases para trabajar con objetos de manera más estructurada.
    • Promesas para trabajar con operaciones asíncronas.
    • Desestructuración de objetos y arrays.
  • ES7 (2016): Fue una actualización pequeña pero útil:
    • Array.prototype.includes(): para verificar si un valor está dentro de un array.
    • Exponentiation operator (**): para hacer potencias más fácilmente.
  • ES8 (2017): Introdujo algunas funcionalidades interesantes para mejorar la programación asíncrona y manipulación de strings:
    • Async/Await: para manejar promesas de manera más sencilla.
    • Object.values() y Object.entries(): para obtener valores y entradas de un objeto.
    • String padding: para agregar espacios o caracteres a la izquierda o derecha de una cadena.
  • ES9 (2018): Introdujo características para mejorar el trabajo con arrays y objetos:
    • Spread operator: para expandir arrays o objetos.
    • Rest parameters: para obtener un número variable de parámetros en funciones.
  • ES10 (2019): Mejoró el manejo de strings, arrays y objetos:
    • Array.prototype.flat() y Array.prototype.flatMap(): para aplanar arrays.
    • String.prototype.trimStart() y String.prototype.trimEnd(): para eliminar espacios al inicio o final de una cadena.
    • Opject.fromEntries(): para convertir un array de pares clave-valor en un objeto.
  • ES11 (2020): Agregó funcionalidades útiles para trabajar con strings y arrays:
    • BigInt: para trabajar con números enteros grandes.
    • Array.prototype.flatMap(): para combinar map() y flat() en un solo paso.
    • String.prototype.matchAll(): para obtener todas las coincidencias de una expresión regular.
  • ES12 (2021) - ES13 (2022): Estas versiones han seguido mejorando la eficiencia y agregando más pequeñas funcionalidades, pero la gran mayoría de los cambios han sido poco visibles para los programadores comunes. Sin embargo, cada nueva versión sigue mejorando la performance y seguridad del lenguaje.
